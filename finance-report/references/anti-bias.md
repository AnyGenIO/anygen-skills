# 反偏见框架

## 何时使用

在以下场景中**必须**执行反偏见检查：
- 建仓 / 加仓决策前
- 财报分析得出结论后
- 论点评估（thesis check）时
- 深度研究报告的最后一步

## 一、6 大认知陷阱自检

每次决策前逐条过：

| 陷阱 | 自检问题 | 危险信号 |
|------|---------|---------|
| **确认偏误** | 我是不是只找了支持买入的数据？ | 搜索词全是正面的、忽略了负面新闻 |
| **锚定效应** | 我是不是死守某个目标价/成本价？ | "分析师说$XX所以一定会到" |
| **处置效应** | 赚了想跑、亏了不走？ | 浮盈就焦虑、浮亏反而"再等等" |
| **近因偏误** | 最近一条新闻是不是权重太大？ | 一条利好就想全仓、一条利空就想清仓 |
| **从众效应** | 是不是因为"大家都在买"？ | 论据全是"分析师一致看好"而非自己的分析 |
| **过度自信** | 是不是觉得自己不可能错？ | 没有设止损、仓位超过合理上限 |

**输出格式**：对每个陷阱标注 ✅ 通过 / ⚠️ 有苗头 / 🚨 触发

## 二、7 大财务红旗

| 红旗 | 检查方法 |
|------|---------|
| 应收账款增速 > 收入增速 | 历史财务对比 AR 和 Revenue 趋势 |
| 存货异常堆积 | 存货增速 vs 收入增速 |
| 经营现金流持续低于净利润 | OCF vs Net Income |
| 频繁变更会计准则 | SEC 文件检查 risk excerpt |
| 关联方交易异常 | SEC 文件检查 8-K |
| 管理层大额抛售（非计划性） | 内部人交易检查近期交易 |
| 审计师更换 / 非标意见 | SEC 文件检查年报 |

## 三、常见盲区自检

| 盲区 | 问自己 |
|------|--------|
| TAM 幻觉 | 市场真有这么大吗？谁在实际付费？ |
| 客户集中度 | 前 3 大客户占比多少？丢一个会怎样？ |
| 护城河虚实 | 换供应商的成本到底有多高？有没有第二供应商？ |
| 增速可持续性 | 当前增速是一次性订单还是持续需求？ |
| 估值脱钩 | 如果增速回归均值，现在的估值还合理吗？ |

## 四、Pre-Mortem（事前尸检）

**在建仓/加仓前必做。**

> 假设 6 个月后这笔交易亏了 30%。最可能的 3 个原因是什么？

1. ___
2. ___
3. ___

然后检查：
- 这些原因现在有没有苗头？
- 如果有苗头，还值得建仓吗？
- 止损位是否覆盖了这些风险？

## 五、结论覆盖

完成反偏见检查后，在报告/决策中加入一段：

```
### 反偏见检查
- 认知陷阱: X/6 通过 (或标出哪些有苗头)
- 财务红旗: X/7 触发 (或标出哪些触发)
- Pre-Mortem 风险: [列出前3个风险及当前状态]
- 结论: 决策未受偏见影响 / 存在 XX 偏见需警惕
```
