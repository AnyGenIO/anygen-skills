# Variant View（变异视角）

## 什么是 Variant View

市场共识往往是错的。Variant View 就是找出**市场定价中隐含的假设，然后检验这些假设是否正确**。

如果你的判断和市场一致，你只能赚平均回报。超额回报来自**你对了而市场错了**的那些点。

## 框架

### 第一步：提取市场共识

从以下来源提取市场当前隐含的假设：
- 当前估值隐含的增速（用 DCF 反向推导）
- 分析师一致预期
- Put/Call 比率和期权隐含波动
- 机构持仓变化方向

### 第二步：列出市场隐含假设

例如：
```
市场对 XX 的隐含假设：
1. 收入增速将从 XX% 回落至 ~XX%（FwdPE XX 隐含）
2. 客户集中度风险已充分定价
3. 竞争威胁不会颠覆核心业务
4. 行业景气周期至少持续到 XXXX 年
```

### 第三步：逐条验证 — 市场对了还是错了？

| 隐含假设 | 你的判断 | 依据 [T1/T2/T3] | 如果市场错了 → |
|---------|---------|----------------|--------------|
| ... | ... | ... | ... |

### 第四步：Variant Perception 结论

```
我的 Variant View:
市场低估了 [XX]，高估了 [YY]。
净效果：[利多 / 利空 / 中性]
Alpha 来源：[具体哪个假设差异最大]
```

## 何时使用

- 首次覆盖 → 必做
- 财报分析 → 建议做（检查市场反应是否合理）
- 赛道扫描 → 可选（找最被低估的标的）
