# 财报分析框架

标的发布财报后，按此框架执行专业级分析。

## 第一步：Beat/Miss 量化

```
Revenue:  $实际M vs $预期M → beat/miss $差额M (+X.X%)
EPS:      $实际 vs $预期 → beat/miss $差额 (+X.X%)
毛利率:   实际% vs 预期% → ±Xbps
指引:     Q+1 $实际M vs $预期M → 上调/维持/下调
FY 指引:  $实际M vs $预期M → 上调/维持/下调
```

## 第二步：Beat 质量评估

不是所有 beat 都一样。区分：

| 类型 | 信号 | 行动 |
|------|------|------|
| 有机 beat + 指引上调 | 最强——需求真正超预期 | 加仓信号 |
| 有机 beat + 指引维持 | 不错，但管理层保守 | 持有，观察下季度 |
| Beat 靠一次性因素 | 假信号——税率、汇率、会计 | 不行动 |
| Revenue beat + margin miss | 增长靠牺牲利润 | 警惕，检查定价权 |
| Miss + 指引上调 | 短期压力但前景改善 | 可能是买入机会 |
| Miss + 指引下调 | Kill switch——论点可能失效 | 减仓/清仓 |

## 第三步：关键指标趋势（4季度滚动）

构建趋势表：

```
指标          Q-3    Q-2    Q-1    Q0(本季)  趋势
Revenue YoY%  ___    ___    ___    ___       加速/减速/稳定
毛利率%       ___    ___    ___    ___       扩张/收缩/稳定
营运利润率%   ___    ___    ___    ___       扩张/收缩/稳定
FCF Margin%   ___    ___    ___    ___       改善/恶化
收入增速      ___    ___    ___    ___       逐季加速=最强信号
```

**关键判断**：收入增速是否在加速？这是成长股最重要的信号。

## 第四步：管理层评论解读

先获取财报后新闻/卖方解读，再从 earnings call 中提取（用搜索找 transcript 摘要）：

1. **需求信号**：backlog、pipeline、订单趋势
2. **定价权**：ASP 变化、折扣压力
3. **客户动态**：新客户获取、大客户扩展、流失率
4. **竞争格局**：win rate 变化、competitive displacement
5. **资本开支意图**：扩产、研发投入方向
6. **宏观敏感度**：管理层对经济环境的评价

**红旗词汇**（出现即警惕）：
- "challenging environment"、"headwinds"、"cautious"
- "elongated sales cycles"、"delays"
- "restructuring"、"cost optimization"（非增长期的说法）

**绿旗词汇**（正面信号）：
- "ahead of plan"、"accelerating"、"record"
- "strong pipeline"、"expanding TAM"
- "supply constrained"（需求太强供不上）

## 第五步：市场反应分析

```
盘后/盘前反应: +/- X%
Day 1 收盘: +/- X%  (相对开盘)
历史反应对比:
  Q-1: +/- X% (beat/miss)
  Q-2: +/- X% (beat/miss)
  Q-3: +/- X% (beat/miss)
```

用历史价格拉取财报日前后真实价格/成交量数据，避免只看新闻叙事。

判断：反应是否合理？过度反应 = 机会。

## 第六步：论点影响评估

对照持仓论点的每个 Pillar：

| Pillar | 本季数据 | 强化/削弱/中性 | 说明 |
|--------|---------|---------------|------|
| Pillar 1 | | | |
| Pillar 2 | | | |
| Pillar 3 | | | |

Kill Switch 检查：
- [ ] 收入低于预告/指引？
- [ ] 利润率大幅收缩？
- [ ] 客户流失或竞争恶化？
- [ ] 管理层变动？

## 第七步：行动决策

基于以上分析，输出明确行动：

| 情景 | 条件 | 行动 |
|------|------|------|
| 强化 | Beat + 指引上调 + Pillar 强化 | 加仓（按推演方案） |
| 持有 | Beat/持平 + 指引维持 | 保持仓位 |
| 减仓 | Miss 或 指引下调 | 减仓 |
| 清仓 | Kill Switch 触发 | 执行清仓 |
